## Задача 1. Подключение к БД и простой SELECT

**Цель:** научиться подключаться к PostgreSQL из Go-кода через `database/sql` (или `pgx` через `stdlib`), проверять соединение и выполнять простой запрос.

### Описание

Реализуй небольшое консольное приложение в файле `cmd/api/main.go`, которое:

1. Читает строку подключения к БД из **переменной окружения** (или конфига), например:
   - `DB_DSN=postgres://user:password@localhost:5432/myshop?sslmode=disable`
2. Создаёт объект `*sql.DB` (или пул `*pgxpool.Pool`) и выполняет `Ping`, чтобы убедиться, что БД доступна.
3. Выполняет запрос:
   ```sql
   SELECT 1;
   ```
4. Читает результат и выводит его в консоль в понятном виде, например:
   ```text
   DB ping OK, SELECT 1 result: 1
   ```

Если подключиться к БД не удалось, или запрос вернул ошибку, приложение должно вывести сообщение об ошибке и завершиться с ненулевым кодом.

### Требования

- Весь код для этой задачи должен находиться в папке `cmd/api/` (минимум файл `main.go`).
- Строка подключения **не должна быть захардкожена** в коде — используй переменную окружения или конфиг.
- Обязательно обрабатывай ошибки:
  - при разборе строки подключения;
  - при создании соединения;
  - при `Ping`;
  - при выполнении `SELECT 1`.
- Используй `context.Context` при работе с БД (по возможности).

### Подсказки

- После создания `*sql.DB` не забудь в конце `main` сделать:
  ```go
  defer db.Close()
  ```

### Критерии готовности

- Приложение успешно подключается к БД (Ping проходит без ошибки).
- При запуске видно понятный лог/сообщение об успешном подключении и результате `SELECT 1`.
- При неправильном DSN или недоступной БД приложение корректно выводит ошибку и не паникует.
